# HEARTBEAT.md - 心跳检查清单

**心跳频率**: 每 30-60 分钟  
**检查时间**: 白天活跃时段 (08:00-23:00)

---

## 📋 心跳检查清单

### 每次心跳检查 (轮询执行)

- [ ] **邮箱检查** - 查看紧急未读邮件
- [ ] **日历检查** - 未来 24-48 小时事件
- [ ] **天气检查** - 如果用户可能外出
- [ ] **记忆审查** - 检查是否需要更新 MEMORY.md

### 每日至少一次

- [ ] **审查 .learnings/** - 处理新的学习/错误/功能请求
- [ ] **更新 memory/YYYY-MM-DD.md** - 记录当日重要事件
- [ ] **检查工作区状态** - git status, 未完成的任务

### 每周一次 (周一上午)

- [ ] **审查 MEMORY.md** - 移除过时信息，添加重要内容
- [ ] **审查技能列表** - 更新或删除不需要的技能
- [ ] **系统更新检查** - openclaw 版本检查

---

## 🧠 记忆维护流程

### 何时更新 MEMORY.md

**应该添加**:
- 用户明确说"记住这个"
- 重复出现的偏好或习惯
- 重要的项目/人物/日期信息
- 长期有效的决策和原则

**应该移除**:
- 过时的项目信息
- 已完成的临时任务
- 不再适用的偏好
- 重复或冲突的内容

### 记忆提升标准

从 `memory/YYYY-MM-DD.md` 提升到 `MEMORY.md`:

| 标准 | 说明 |
|------|------|
| **重复性** | 出现 3 次以上的模式 |
| **重要性** | 影响长期决策或行为 |
| **持久性** | 不是临时状态，会持续存在 |
| **通用性** | 适用于多个场景，不是单一事件 |

## ⏰ 心跳响应规则

**回复 HEARTBEAT_OK 当**:
- 深夜 (23:00-08:00) 且无紧急事项
- 刚检查过 (<30 分钟)
- 无新发现或待办事项
- 用户明显忙碌中

**主动报告当**:
- 发现紧急邮件或消息
- 日历事件即将开始 (<2 小时)
- 完成重要任务
- 超过 8 小时未交流
