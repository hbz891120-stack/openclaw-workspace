## 🎯 优化措施详解

### 1️⃣ 切换到快速模型

**qwen3.5-flash** vs **qwen3.5-plus**:

| 特性 | qwen3.5-flash | qwen3.5-plus |
|------|---------------|--------------|
| 响应速度 | ⚡⚡⚡ 极快 | ⚡⚡ 中等 |
| 准确度 | ⭐⭐⭐⭐ 良好 | ⭐⭐⭐⭐⭐ 优秀 |
| 适用场景 | 日常对话、简单任务 | 复杂推理、代码生成 |
| Token 成本 | 低 | 中等 |

**切换命令**:
```bash
# 临时切换到快速模型
/model qwen3.5-flash

# 临时切换到强力模型
/model qwen3.5-plus
```

### 2️⃣ 激进的上下文压缩

**compaction.mode: aggressive**
- 自动压缩旧对话
- 保留关键信息
- 减少上下文长度
- 加快响应速度

**压缩策略**:
| 模式 | 触发条件 | 压缩程度 |
|------|----------|----------|
| safeguard | 接近上限时 | 最小 |
| aggressive (新) | 主动压缩 | 中等 |

### 3️⃣ 提高并发数

**maxConcurrent: 4 → 8**
- 同时处理更多请求
- 减少排队等待
- 提升整体吞吐量

**subagents.maxConcurrent: 8 → 16**
- 子代理并行度翻倍
- 复杂任务更快完成

## ⚙️ 其他优化建议

### 1. 减少上下文长度
- 定期开启新会话
- 避免过长的历史对话
- 使用 `/compact` 手动压缩

### 2. 避免重复查询
- 相同问题尽量在同一会话
- 利用记忆系统减少重复说明

### 3. 合理使用子代理
- 简单任务不用子代理
- 复杂任务才 spawn 子代理
- 设置合理的 timeout

### 4. 网络优化
- 确保网络稳定
- 使用国内 API 端点 (已配置)
- 避免高峰时段大任务
