## 🎯 修复原则

### 避免 "Unsupported channel" 错误
1. **不指定 channel**: delivery 配置中只设置 `mode: "announce"`，不指定 `channel` 或 `to`
2. **使用默认模式**: 让系统自动选择可用的通知方式
3. **避免硬编码**: 不在 cron 配置中硬编码特定 channel (如 whatsapp/telegram)

### 避免超时错误
1. **合理设置超时**: 根据任务复杂度设置足够的时间
   - 简单任务：60-120 秒
   - 中等任务：300-600 秒
   - 复杂任务：600-900 秒
2. **考虑网络因素**: 外部 API 调用需要更多缓冲时间
3. **监控执行时间**: 根据实际执行情况调整
