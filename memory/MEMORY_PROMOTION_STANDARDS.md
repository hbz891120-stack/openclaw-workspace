# 记忆提升标准

**用途**: 定义何时将内容从每日记忆提升到长期记忆

---

## 📊 记忆架构

```
每日记忆 (memory/YYYY-MM-DD.md)
         ↓
    [审查筛选]
         ↓
长期记忆 (MEMORY.md)
```

---

## ✅ 提升标准 (必须满足至少 2 项)

### 1️⃣ 重复性 (Recurring)
- 同一主题出现 **3 次以上**
- 用户多次提及的偏好
- 反复出现的问题或模式

**示例**:
- ✅ 用户 3 次提到喜欢简洁回复
- ❌ 单次提到的临时偏好

### 2️⃣ 重要性 (Important)
- 影响长期决策
- 改变工作方式
- 涉及重要关系或项目

**示例**:
- ✅ 用户的工作项目截止日期
- ❌ 今天午餐吃了什么

### 3️⃣ 持久性 (Persistent)
- 不是临时状态
- 预计持续 >1 个月
- 稳定的偏好或习惯

**示例**:
- ✅ 用户的时区和语言偏好
- ❌ 用户今天心情不好

### 4️⃣ 通用性 (General)
- 适用于多个场景
- 不是单一事件
- 可复用的知识

**示例**:
- ✅ 用户的沟通风格偏好
- ❌ 某次会议的具体内容

---

## 🚫 不应该提升的内容

- 临时状态 (心情、天气、一次性事件)
- 过时的项目信息
- 已经完成的待办事项
- 重复或冲突的内容
- 过于细节的对话记录

---

## 📋 提升流程

### 步骤 1: 审查每日记忆
```bash
# 查看今日记忆
cat memory/$(date +%Y-%m-%d).md

# 查看最近 7 天记忆
for f in memory/2026-03-*.md; do cat $f; done
```

### 步骤 2: 识别候选内容
寻找符合提升标准的内容：
- 用户明确说"记住这个"
- 重复出现的主题
- 重要的决定或承诺

### 步骤 3: 提炼和添加
将候选内容提炼为简洁的规则或事实，添加到 MEMORY.md 适当章节。

### 步骤 4: 标记和记录
在每日记忆中标注已提升的内容：
```markdown
<!-- 已提升至 MEMORY.md - 2026-03-18 -->
```

---

## 🔄 审查周期

| 周期 | 操作 |
|------|------|
| **每日** | 更新 memory/YYYY-MM-DD.md |
| **每周** | 审查并提升重要内容 |
| **每月** | 清理过时内容 |

---

## 📝 示例

### 每日记忆中的原始记录
```markdown
## 2026-03-18

用户说以后所有搜索都用 Tavily，不要用 Brave Search。
配置了 TAVILY_API_KEY 到 ~/.openclaw/.env。
测试了搜索功能，工作正常。
```

### 提升到 MEMORY.md (提炼后)
```markdown
## 搜索引擎
- **默认**: Tavily Search
- **禁用**: Brave Search
- **配置**: ~/.openclaw/.env
```

---

## 💡 最佳实践

1. **简洁优先** - MEMORY.md 应该是要点列表，不是详细日志
2. **结构化** - 使用清晰的分类和标题
3. **可操作** - 内容应该能指导未来行为
4. **定期清理** - 移除过时或不再适用的内容
5. **版本意识** - 重大变更时记录日期

---

## 🎯 质量检查清单

提升前问自己：
- [ ] 这条信息 1 个月后还有用吗？
- [ ] 这适用于多个场景吗？
- [ ] 这能指导我未来的行为吗？
- [ ] 这是用户希望我记住的吗？
- [ ] 这足够简洁吗？

全部 ✅ 才提升！
