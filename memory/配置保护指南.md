# 🛡️ OpenClaw 配置保护指南

**创建时间**: 2026-03-19  
**目的**: 避免 `openclaw.json` 配置错误导致无法启动

---

## 📋 问题原因

`openclaw.json` 是 OpenClaw 的核心配置文件，以下情况可能导致损坏：

1. **手动编辑错误** - JSON 格式错误 (缺少逗号、引号等)
2. **并发写入** - 多个进程同时修改
3. **意外中断** - 编辑过程中断电/崩溃
4. **工具 Bug** - 配置工具写入错误数据

---

## ✅ 解决方案

### 1️⃣ 自动备份系统

**位置**: `~/.openclaw/scripts/backup-config.sh`

**功能**:
- 手动备份配置
- 恢复历史备份
- 验证 JSON 格式
- 自动清理旧备份 (保留 10 个)

**用法**:
```bash
# 备份当前配置
~/.openclaw/scripts/backup-config.sh backup

# 查看可用备份
~/.openclaw/scripts/backup-config.sh list

# 恢复配置 (先列出，再选择)
~/.openclaw/scripts/backup-config.sh restore
~/.openclaw/scripts/backup-config.sh restore openclaw.json.20260319_105400.gz

# 验证当前配置
~/.openclaw/scripts/backup-config.sh validate
```

---

### 2️⃣ 安全编辑工具

**位置**: `~/.openclaw/scripts/safe-config-edit.sh`

**功能**:
- 编辑前自动备份
- 编辑后自动验证
- 格式错误自动回滚

**用法**:
```bash
# 安全编辑配置
~/.openclaw/scripts/safe-config-edit.sh
```

**流程**:
1. 验证当前配置
2. 创建临时备份
3. 打开编辑器 (nano/vim)
4. 保存后验证格式
5. 错误则提示回滚

---

### 3️⃣ 自动备份 Cron

**位置**: `~/.openclaw/scripts/auto-backup-on-change.sh`

**功能**:
- 每小时检测配置变化
- 变化时自动备份
- 基于 checksum 判断

**Cron 任务**:
```bash
# 每小时自动备份
0 * * * * ~/.openclaw/scripts/auto-backup-on-change.sh
```

---

### 4️⃣ 备份目录结构

```
~/.openclaw/config-backups/
├── openclaw.json.20260319_105400.gz   # 手动备份
├── openclaw.json.20260319_115500.gz   # 手动备份
├── openclaw.json.auto.20260319_120000.gz  # 自动备份
├── openclaw.json.auto.20260319_130000.gz  # 自动备份
└── .config-checksum                     # checksum 记录
```

## 🔧 紧急恢复流程

### 情况 1: Gateway 无法启动

```bash
# 1. 验证配置
~/.openclaw/scripts/backup-config.sh validate

# 2. 查看可用备份
~/.openclaw/scripts/backup-config.sh list

# 3. 恢复最近的备份
~/.openclaw/scripts/backup-config.sh restore

# 4. 重启 Gateway
openclaw gateway restart
```

### 情况 2: 编辑后配置错误

```bash
# 安全编辑工具会自动提示回滚
# 如果跳过回滚，手动恢复:
cp ~/.openclaw/config-backups/openclaw.json.pre-edit.*.gz /tmp/
gunzip /tmp/openclaw.json.pre-edit.*.gz
cp /tmp/openclaw.json.pre-edit.* ~/.openclaw/openclaw.json
```

## 📝 最佳实践

### ✅ 推荐做法

1. **编辑前备份**
   ```bash
   ~/.openclaw/scripts/backup-config.sh backup
   ```

2. **使用安全编辑工具**
   ```bash
   ~/.openclaw/scripts/safe-config-edit.sh
   ```

3. **重大变更前导出**
   ```bash
   cp ~/.openclaw/openclaw.json ~/Desktop/openclaw.backup.json
   ```

4. **定期验证配置**
   ```bash
   ~/.openclaw/scripts/backup-config.sh validate
   ```

5. **使用 Git 版本控制** (可选)
   ```bash
   cd ~/.openclaw
   git init
   git add openclaw.json
   git commit -m "Initial config"
   ```

---

### ❌ 避免做法

1. **不要直接用文本编辑器修改**
   ```bash
   # ❌ 危险
   nano ~/.openclaw/openclaw.json
   
   # ✅ 安全
   ~/.openclaw/scripts/safe-config-edit.sh
   ```

2. **不要同时多处编辑**
   - 避免在多个终端同时编辑
   - 避免手动编辑 + 工具编辑同时进行

3. **不要跳过验证**
   - 任何修改后都要验证 JSON 格式

4. **不要删除备份目录**
   - `~/.openclaw/config-backups/` 是救命稻草

## 🔍 配置验证命令

```bash
# 快速验证 JSON 格式
python3 -m json.tool ~/.openclaw/openclaw.json > /dev/null && echo "✅ 有效" || echo "❌ 错误"

# 查看配置大小
ls -lh ~/.openclaw/openclaw.json

# 查看配置修改时间
stat ~/.openclaw/openclaw.json

# 查看配置 checksum
md5sum ~/.openclaw/openclaw.json
```
